Google Dorks :-


GoogleDorking is a special searching technique to query google search engine for information hidden on public websites and vulnerabilities exposed by public servers. It makes the life of a hacker more easy .


Available dorks

  1. site :

    Search for the result in the given domain

  2. related :

    Search for Similar web pages

  3. cache :

    Display the web pages stored in Cache

  4. link :

    List the websites having a link to a specific web page

  5. alligntext :

    Search for websites containing a specific keyword

  6. intext :

    Search for documents containing a specific keyword

  7. allintitle :

    Search for websites containing a specific keyword in the title

  8. intitle :

    Search for documents containing a specific keyword in the title

  9. allinurl :

    Search for websites containing a specific keyword in URL

  10. inurl :

    Search for documents containing a specific keyword in URL

Example

  • suppose you want to search websites running php and want to test xss vulnerability in it you can make a simple google search with inurl dork .

    inurl=".php"

Google Hacking database (GHDB) :-


The Google Hacking Database (GHDB) is a part of Google hacking search terms that have been found to reveal sensitive data exposed by vulnerable servers and web applications. The GHDB was launched in 2000 by Johnny Long to serve penetration testers. In 2010, Long turned the database over to Offensive Security and it became part of exploit-db.com . It was also expanded to include not only the Google search engine but also other search engines like Microsoft’s Bing as well as other repositories such as GitHub.

categories of queries in GHDB 


  • Error messages that contain sensitive information such as directory paths
  • Files with sensitive data, passwords, and user names
  • exploits for version specific web services
  • Product-specific advisories
  • Detailed information about web servers
  • Sensitive online shopping data
